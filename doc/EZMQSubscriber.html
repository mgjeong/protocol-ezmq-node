Class: EZMQSubscriber

EZMQSubscriber(ip, port, subCB, subTopicCB)

This class represent EZMQSubscriber. It contains APIs to start/stop subscriber and to subscribe for data.

ProtoType of subCB :- function subCallback(event) {}
ProtoType of subTopicCB :- function subTopicCallback(topic, event){}

In callback event will be EZMQEvent/EZMQByteData

subscribe(topic) → {EZMQErrorCode}

Subscribe for event/messages. This API can be called to subscribe for all events or subscribe events for given topic/s.
Note:
(1) If API is called without topic, it will subscribe for all the events.
(2) Topic can be a single topic [String] or list of topic [String list].
(3) Topic name should be as path format. For example: home/livingroom/
(4) Topic name can have letters [a-z, A-z], numerics [0-9] and special characters _ - . and /
Parameters:
Name Type Description
topic string/stringlist Subscribe for given topic. It can be empty.

subscribeWithIPPort(ip, port, topic) → {EZMQErrorCode}

Subscribe for event/messages from given IP:Port on the given topic.
Note:
(1) It will be using same Subscriber socket for connecting to given ip:port.
(2) To unsubcribe use un-subscribe API with the same topic.
(3) Topic name should be as path format. For example: home/livingroom/
(4) Topic name can have letters [a-z, A-z], numerics [0-9] and special characters _ - . and /
(5) Topic will be appended with forward slash [/] in case, if application has not appended it
(6) If using in secured mode: Call setServerPublicKey(serverPublicKey) API with target server public key before calling this API.
Parameters:
Name Type Description
ip string Target[Publisher] IP address.
port number Target[Publisher] Port number.
topic string Topic to be subscribed.
